# AGENTS — Convention entry point

If you're an AI coding tool (Claude Code, Cursor, GitHub Copilot, Aider, Cline, or anything else) opening this repo for the first time, **read these four files in this order**:

1. [`BODY.md`](BODY.md) — the live infrastructure map (autogenerated from the codebase). Tells you what actors exist, what scrapers exist, what self-awareness components exist, what dependencies exist, what env vars are read, what the public API surface is. **Always read this first.**

2. [`concepts/the_house.md`](concepts/the_house.md) and [`concepts/self_awareness_loop.md`](concepts/self_awareness_loop.md) — the conceptual model the codebase implements. If you're going to make structural changes, you need to know the metaphor.

3. [`LESSONS.md`](LESSONS.md) — structural mistakes the project has already burned hours on. The "why" behind the conventions BODY.md describes. Read before refactoring.

4. [`docs/stack/`](docs/stack/) — one file per external dependency, explaining what it does and why we picked it. Read the relevant one before importing a new package.

## What stays consistent across tools

- **One source of truth.** BODY.md is *the* live map. The per-tool pointer files (this `AGENTS.md`, `CLAUDE.md`, `.cursor/rules/m3xa.mdc`, `.github/copilot-instructions.md`) all funnel you back here.
- **Auto-maintained.** Don't hand-edit the sections of BODY.md between `<!-- AUTOGEN -->` markers — they regenerate from the code. Edit the code, then commit; the pre-commit hook updates BODY.md.
- **CI is the airbag.** Pre-commit can be bypassed with `--no-verify`, but `.github/workflows/body_in_sync.yml`, `stack_in_sync.yml`, and `anonymization_check.yml` all re-run on push and block the merge.

## What's anonymized

This is a **public, didactic** reference. The system it descends from has real source names (banks, expert analysts, podcasts, wire services) that **never appear in this repo**. They are replaced with `Bank1`, `Expert1`, `Podcast1`, etc. See [`docs/source_naming.md`](docs/source_naming.md).

**Before opening a PR, run:**

```bash
python tools/check_anonymization.py
```

CI will reject anything that introduces a real name. If you're contributing scraper templates, expertise files, or example queries, double-check.

## Layered enforcement

When you change something that affects the autogen sections, three layers verify you regenerated BODY.md:

```
edit → pre-commit hook → CI body_in_sync → tests/test_body_in_sync.py
```

Same shape for the stack registry and the anonymization check. See `concepts/self_knowledge_layer.md` for the rationale.

## When stuck

If BODY.md says one thing and the code says another, **trust the code and update BODY.md** — that's the whole point of the autogen layer. Run:

```bash
python tools/regenerate_body.py
```

Then commit the updated BODY.md alongside your change.
