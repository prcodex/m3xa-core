# Self-knowledge layer

> The docs decay or they don't. With autogen + CI enforcement, they don't.

Stub. Full essay covers:

- The autogen pattern: `<!-- AUTOGEN:start:name -->` markers in BODY.md, regenerated by `tools/regenerate_body.py`
- Three-layer enforcement: pre-commit (fast, bypassable) + CI (unbypassable, slower) + pytest mirror (catches it locally before push)
- The `_strip_meta` trick — content-diff before updating the timestamp, otherwise every run is "out of sync"
- Why a single BODY.md beats a directory of body docs (single doorway for all coding tools)
- The stack registry as a sibling pattern: every external dep gets a `docs/stack/<name>.md`, CI fails on missing docs
- The anonymization check as a third sibling: forbidden tokens hashed in `.anonymization-blocklist.sha256`
